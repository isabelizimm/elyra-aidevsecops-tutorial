Pre-requisites¶

In this section, the user can find the requirements that should be available before starting the tutorial.

Login¶

Operate First Login¶

Using Operate First you just need:

  • Gmail account required to login to JupyterHub.

Other environments Login¶

For other environment you might receive Openshift credentials.

GitHub account¶

The project is based on GitHub, if you don’t have one just following this link.

GitHub token¶

If you don’t have a GitHub token, you can create one following GitHub docs: create GitHub token.

Namespace for deployment¶

Operate First namespace for deployment¶

Using Operate First you can request access to a namespace where you want to deploy your application using this link.

Other environments namespace for deployment¶

If you have a different environment, you usually receive one by default or you need to ask admins to create a namespace for you.

Cloud object storage¶

Operate First Cloud object storage¶

Using Operate First you can request a new bucket once you get a namespace. In order to do that, use the following link.

Once you created a new bucket in your namespace, you can login in the cluster using the following link and in your namespace you will find:

  • AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEYin a Secret Object with the bucket name you crated.

  • OBJECT_STORAGE_ENDPOINT_URL, OBJECT_STORAGE_BUCKET_NAME in a ConfigMap Object with the bucket name you crated.

Other Cloud object storages¶

You can use your own bucket credentials, setting them as env variables AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY. You also need to have OBJECT_STORAGE_ENDPOINT_URL, OBJECT_STORAGE_BUCKET_NAME.
